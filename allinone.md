## Summary of all searched results

### 1. Overview

6 个查询并集（IVM/物化视图、self-adjusting computation、partial evaluation、typed effects/effect handlers、rewriting-using-views、增量重算一致性/demand-driven 分析），1995–2026，在 arXiv + Crossref + DBLP 三源共得 **131 篇去重论文**（9 条跨源重复合并）。语料清晰覆盖三条学术谱系——数据库的增量视图维护、PL 的自调节计算与部分求值、以及 2009 年后的代数效应——并在 2013–2026 出现系统侧的现代化复兴（DBSP、differential dataflow 谱系）。

### 2. Trends

- **三个时代**：1995–2005 是 IVM 黄金时代（SIGMOD/VLDB/ICDE/TKDE，Rundensteiner、Gupta/Mumick、Griffin/Libkin 等），核心产出是 delta 规则与维护算法分类学；2002–2014 是 PL 时代（Acar 学派的 self-adjusting computation、Jones/Glück 学派的 partial evaluation）；2009–2023 是代数效应时代（Plotkin/Pretnar → Leijen → Brachthäuser/Ostermann）。
- **2013 后的系统复兴**：differential dataflow [M3] → DBSP [9] → Noria [M4] → Enzyme [4]（2026），IVM 从数据库内核问题变成流处理/数据工程基础设施问题；同时程序分析被重新表述为 IVM（IncA [M5]）。
- **重复出现的团队**：Christoph Koch 学派（[1][7][8] + DBToaster [M9] + LINVIEW）、Umut Acar 学派（[23][72][73][75] + Adapton [M2]）、Rundensteiner 组（[14][19][21]）、Klaus Ostermann 组（[3][76]）。
- **venue 分布**：DBLP 贡献了 1998–2012 的 VLDB/WWW/DOLAP/ICDE 老论文；Crossref 贡献了高引用的 POPL/PLDI/ICFP 论文；arXiv 噪声最大（量子计算、高斯泼溅、词典型条目约占尾部 1/4）。
- **一个值得注意的空洞**："red-green"与"stable names"几乎不在正式文献中——它们活在 Roslyn 设计博客、Salsa/rust-analyzer、Shake/Gradle 等工业增量编译器/构建系统文档里；学术界对应的可引用载体是 Build Systems à la Carte [M8] 与 Adapton [M2]。

### 3. Key themes（按"可整体迁移的实验范式"组织）

1. **IVM 经典范式：delta 化 + from-scratch consistency 检验** — 正确性 = 增量维护结果 ≡ 全量重算；实验 = 更新负载规模 × 视图复杂度扫描，比较 delta 维护 vs 重算。代表：[1][7][8][9][14][21][22][37][45][87][96][131]；[M7][M9]。这是与你目标最同构的范式：**有效性判据（≡ from-scratch）和实验设计（双跑对照）可以整套搬走**。
2. **Self-adjusting computation 范式：edit-trace 基准 + 双跑对照** — 在合成/真实编辑序列上跑 change propagation，以 from-scratch 执行为 oracle 验证等价并测加速比；记忆化（stable names 的学术版 = modifiable references + trace）是核心机制。代表：[23][72][73][75][74][103]；[M1][M2][M8]。
3. **Query-relative equivalence / answering-using-views** — "等价"被定义为相对于可用视图集合的可重写性，而非绝对语义等价；determinacy 理论 [42][83] 给出何时视图足以回答查询的判据。代表：[6][40][41][42][47][48][58][59][62][83]；[M6]。这是"consumer-specific projections"的理论原型：每个 consumer 拿到的是相对其查询接口的投影，有效性 = 投影对该 consumer 查询封闭。
4. **Partial evaluation / specialization 范式：Futamura 一致性** — residual program 在剩余输入上 ≡ 原程序（另一种 from-scratch consistency）；实验 = specialization 前后在固定/变化输入划分下的性能与正确性双跑。代表：[2][16][17][26][38][66][94][97][99][101][114][115][117]。
5. **Typed effects / effect handlers：把"计算触碰了什么"变成类型级证据** — effect 类型静态限定计算可读写的资源，handler 决定拦截语义；这是把"复用有效性条件"类型化的最近邻工具。代表：[3][10][11][18][20][55][69][70][71][76][78]。
6. **Demand-driven / 按需增量分析（red-green 的学术对应物）** — 分析结果按需计算、按变更失效；增量一致性检查。代表：[13][31][95][100][15]；[M5]。

### 7. Recommendations for reading（按"迁移整套实验范式"的目标排序，基础 → 前沿）

1. **Gupta & Mumick [M7] (1995) + Griffin & Libkin [45][96] (1997–99)** — IVM 的问题分类学与 delta 范式起点；from-scratch consistency 作为正确性判据在这里已经成熟，直接可搬。
2. **Adaptive Functional Programming [M1] (2002) + Imperative SAC [23] (2008)** — edit-trace 基准 + 双跑 oracle 的实验设计，是评测"变更后复用"类系统的标准模板。
3. **Answering Queries Using Views [M6] (1995) + Determinacy [42] (2011)** — query-relative equivalence 的形式化：有效性 = 相对 consumer 查询的可重写性，是 consumer-specific projection 的理论根基。
4. **Build Systems à la Carte [M8] (2018)** — 把 from-scratch consistency 写成可检验的 correctness 性质（rebuilder ≡ full build），并系统比较 12+ 种真实系统的增量策略（stable names / red-green 所在的谱系）；其实验矩阵可整套复用。
5. **DBSP [9] (2022) + Noria [M4] (2018)** — 现代化终点：DBSP 给出"增量 ≡ 重算"的代数保证（Z-sets），Noria 给出 consumer-specific 部分物化 + upquery 的系统范式；两者合起来是 2020s 版本的完整参考实现。
