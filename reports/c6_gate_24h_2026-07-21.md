# C6 provenance artifact 复用 · 24h 结构资格实验(GATE)结果备忘录

日期：2026-07-21 · 零模型调用 · 对应《重新开辟研究报告》§13-C6 第一级
目的：验证候选域的**结构性前提 1**——依赖是否天然稀疏、节点特定(NEG-25 纪律:
先于任何 mutation 设计或模型调用)。

## 对象与方法

- 对象：两个真实 Python 仓库——`aaai2027-route-a-plus`(159 模块)与
  `aaai2027`(5747 模块,含大型 pipeline 树)。
- 方法:AST 静态 import 解析 → 模块依赖 DAG → 对每个模块计算单点 mutation 的
  传递爆炸半径(transitive dependents 占全图比例)。
- 预登记判定门槛(计算前写下):
  - G1:median affected-fraction ≤ 0.50
  - G2:≥20% 模块 affected-fraction ≤ 0.10

## 结果

| 指标 | route-a-plus (N=159) | aaai2027 (N=5747) |
|---|---|---|
| 依赖边 | 171 | 5674 |
| median 爆炸半径 | 0.000 | 0.000 |
| p90 | 0.032 | 0.001 |
| 最大爆炸半径(最大 hub) | 0.14(`tracebuild.route_a_plus_v7`) | 0.056 |
| 影响 ≤10% 的模块 | 98.1% | 100.0% |
| 影响 >50% 的模块 | 0% | 0% |
| 叶子模块(无 dependent) | 67.3% | 77.3% |

**判定:GO(G1、G2 均通过,且余量巨大)。** 代码库分析域的依赖结构天然高度稀疏:
单点变更平均只波及全图的 0–3%,不存在"牵一发动全身"的超级 hub。provenance 驱动
复用(未受影响节点直接复用 + 受影响子图 IVM 式增量维护)在该域有结构性空间。

## 如实记录的限制(不得外推)

1. **这只证明了结构性前提 1(域的基底稀疏),没有证明前提 2(workflow 保持稀疏)**。
   NEG-25 的失败正是后者:全上下文 prompt 让 runtime read-set = static,稀疏性在
   workflow 层被人为消灭。一周实验的第一要务是采用**窄上下文路由**(每个 consumer
   节点只喂其依赖切片),并在接入 agent 后重测 static vs runtime read-set——
   这本身是一周实验的内置资格门,若再次 static≡runtime 则按预登记止损。
2. 静态 import 图 ≠ 运行时读取:动态 import、字符串引用、test↔source 约定未建模;
   爆炸半径是保守下界(真实受影响集可能更大,但不可能推翻"稀疏"这一数量级结论)。
3. 仅 Python、仅 2 个仓库;结论限于"代码库分析型 agent workflow"这一类。

## 下一级(一周实验)入口

半合成 producer/consumer DAG:节点 = 对单个模块/文件的真实工具分析(读文件、
提取符号、摘要),边 = import 依赖;每个 consumer 只接收其依赖切片(窄路由);
一个模块被 mutation 后,provenance memoization 只重算受影响子图,与 from-scratch
双跑对照;coverage 强制拆分为 exact 命中与 provenance 增量两个字段(NEG-19 纪律)。
